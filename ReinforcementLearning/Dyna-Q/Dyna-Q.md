# Dyna-Q 算法

Dyna-Q 是 Richard Sutton 于 1991 年提出的一种 **Model-Based** 强化学习框架，核心思想是将 **直接强化学习**（Direct RL）与 **基于模型的规划**（Model-Based Planning）统一起来。前置知识见 [[1.introduction（RL）]]、[[TD]]。

> [!info] Dyna-Q 的定位
> Dyna-Q 是 Model-Free 与 Model-Based 方法的桥梁。它在标准 Q-learning 的基础上增加了一个**环境模型**，利用该模型生成**模拟经验**（Simulated Experience）来加速学习。这样，智能体不仅从真实交互中学习，还从"想象"中学习，大幅提升了**样本效率**。

---

## 背景：Model-Free vs Model-Based

### Model-Free 的局限

[[TD]] 中介绍的 Q-learning、SARSA 等方法都是 **Model-Free** 的：智能体只从真实环境交互中获取经验并更新价值函数。这意味着：

1. **样本效率低**：每条经验只使用一次（或少数几次），大量信息被浪费
2. **交互成本高**：在真实环境中采集数据可能非常昂贵（如机器人控制）
3. **学习速度慢**：完全依赖真实交互，收敛需要大量 Episode

### Model-Based 的优势与代价

Model-Based 方法通过学习一个环境模型 $\hat{p}(s', r | s, a)$，可以在不与真实环境交互的情况下生成模拟经验，从而：

- **提升样本效率**：一条真实经验可以通过模型"放大"为多条模拟经验
- **减少真实交互**：用模型模拟替代昂贵的真实探索

但也有代价：

- **模型偏差**：学习到的模型不可能完美，模型误差会传播到策略中
- **计算开销**：训练模型和生成模拟经验需要额外计算

---

## Dyna 架构

Dyna 架构的核心思想是：**同一套学习算法**既可以作用于真实经验，也可以作用于模型生成的模拟经验。

### 整体框架

```
┌──────────────────────────────────────────────────────┐
│                    Dyna Architecture                 │
│                                                      │
│   ┌─────────┐      真实经验       ┌───────────────┐  │
│   │  环境    │ ───────────────────→│  Q-learning   │  │
│   │ (Real)  │   (s, a, r, s')    │  (Direct RL)  │  │
│   └────┬────┘                     └───────┬───────┘  │
│        │                                  │          │
│        │ 真实经验                    更新 Q(s,a)      │
│        ▼                                  ▲          │
│   ┌─────────┐     模拟经验        ┌───────┴───────┐  │
│   │ 环境模型 │ ──────────────────→│   Planning     │  │
│   │ (Model) │  (ŝ, â, r̂, ŝ')   │   (规划)       │  │
│   └─────────┘                     └───────────────┘  │
└──────────────────────────────────────────────────────┘
```

Dyna 架构包含三个并行的过程：

| 过程 | 输入 | 作用 |
|------|------|------|
| **Direct RL** | 真实经验 $(s, a, r, s')$ | 直接更新 $Q(s, a)$ |
| **Model Learning** | 真实经验 $(s, a, r, s')$ | 学习环境模型 $\hat{M}$ |
| **Planning** | 模型生成的模拟经验 $(\hat{s}, \hat{a}, \hat{r}, \hat{s}')$ | 用模拟经验更新 $Q(s, a)$ |

> [!important] 关键洞察
> Direct RL 和 Planning **使用完全相同的更新公式**（如 Q-learning 更新），唯一区别在于经验的来源——一个来自真实环境，一个来自学习到的模型。

## Dyna-Q 算法详解

### 环境模型

Dyna-Q 中的环境模型 $\hat{M}$ 是一个**确定性模型**，记录每个状态-动作对 $(s, a)$ 对应的下一状态和奖励：

$$
\hat{M}(s, a) = (s', r)
$$

在 **表格型（Tabular）** Dyna-Q 中，模型就是一个查找表，直接存储最近一次观测到的转移。在**深度学习版本**中，模型用神经网络来近似：

$$
\hat{s}', \hat{r}, \hat{d} = f_\theta(s, a)
$$

其中 $f_\theta$ 是参数化的神经网络，分别预测下一状态、奖励和终止信号。

### Q-learning 更新

无论是真实经验还是模拟经验，Q 值的更新公式一致：

$$
Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha \left[ R_{t+1} + \gamma \max_{a'} Q(S_{t+1}, a') - Q(s_t, a_t) \right]
$$

### 规划步骤

每次与真实环境交互后，Dyna-Q 会执行 $n$ 步规划：

1. 从已经历过的 $(s, a)$ 中**随机采样**
2. 用模型查询 $\hat{M}(s, a) \to (\hat{s}', \hat{r})$
3. 用 Q-learning 更新 $Q(s, a)$

参数 $n$（规划步数）控制了**计算量**与**学习速度**之间的权衡。

> [!example]- 伪代码：Tabular Dyna-Q
> **输入**：步长 $\alpha > 0$，折扣因子 $\gamma$，探索率 $\epsilon$，规划步数 $n$
> 1. 初始化 $Q(s, a)$ 和 $\text{Model}(s, a)$，$\forall s \in \mathcal{S}, a \in \mathcal{A}$
> 2. **Loop**（每个 Episode）：
>    1. 初始化 $S$
>    2. **Loop**（每一步）：
>       - (a) $A \leftarrow \epsilon\text{-greedy}(S, Q)$
>       - (b) 执行 $A$，观测 $R, S'$
>       - (c) **Direct RL**：$Q(S, A) \leftarrow Q(S, A) + \alpha [R + \gamma \max_{a'} Q(S', a') - Q(S, A)]$
>       - (d) **Model Learning**：$\text{Model}(S, A) \leftarrow (R, S')$
>       - (e) **Planning**：重复 $n$ 次：
>           - 随机选择之前经历过的 $S_m, A_m$
>           - $R_m, S'_m \leftarrow \text{Model}(S_m, A_m)$
>           - $Q(S_m, A_m) \leftarrow Q(S_m, A_m) + \alpha [R_m + \gamma \max_{a'} Q(S'_m, a') - Q(S_m, A_m)]$
>       - (f) $S \leftarrow S'$

## 规划步数 $n$ 的影响

规划步数 $n$ 是 Dyna-Q 最重要的超参数。Sutton 在经典的迷宫实验中展示了不同 $n$ 值的效果：

| $n$ | 效果 |
|-----|------|
| $n = 0$ | 等价于标准 Q-learning，无规划 |
| $n = 5$ | 显著加速学习 |
| $n = 50$ | 进一步加速，但边际收益递减 |

> [!tip] 经验法则
> 增加规划步数几乎总能加速学习（前提是模型足够准确）。但过大的 $n$ 会增加每步的计算时间，且当模型不准确时可能引入有害偏差。在实践中，通常取 $n \in [5, 50]$。

## 环境变化与 Dyna-Q+

### 模型失效问题

Dyna-Q 有一个根本性弱点：当**环境发生变化**时，之前学到的模型可能不再准确，导致规划产生错误的更新。

两种典型场景：

1. **捷径出现**（Shortcut）：环境中出现了更优的路径，但智能体不会去探索（因为旧模型认为那里不通）
2. **路径阻断**（Blocking）：原有最优路径被封堵，智能体需要重新探索

### Dyna-Q+ 的改进

Dyna-Q+ 通过引入**探索奖励**（Exploration Bonus）来解决模型失效问题。对于每个 $(s, a)$ 对，记录其距上次被真实访问的时间步数 $\tau(s, a)$，在规划时给奖励加上一个额外的探索奖金：

$$
R_{\text{bonus}} = R + \kappa \sqrt{\tau(s, a)}
$$

其中 $\kappa > 0$ 是一个小常数。这意味着**长时间未被访问的状态-动作对会获得更高的奖励**，激励智能体去重新探索它们，从而发现环境的变化。

> [!note] Dyna-Q+ 的直觉
> 如果一个 $(s, a)$ 很久没被访问过了，那么模型对它的预测可能已经过时了。Dyna-Q+ 通过奖励鼓励智能体重新访问这些状态，兼顾了**利用**（exploitation）与**探索**（exploration）。

## 深度 Dyna-Q 实现

在连续状态/动作空间中，表格型方法不再适用。可以使用神经网络替代查找表来实现 Dyna-Q，本项目中的 [[Dyna-Q.py]] 即为深度版本实现，应用于 `Pendulum-v1` 环境。

### 核心组件

| 组件 | 类名 | 功能 |
|------|------|------|
| Q 网络 | `QNetwork` | 估计 $Q(s, a)$，输入状态和动作，输出 Q 值 |
| 目标 Q 网络 | `QNetwork`（副本） | 提供稳定的 TD Target，定期同步 |
| 环境模型 | `ModelNetwork` | 预测 $(\hat{s}', \hat{r}, \hat{d})$，三个独立的输出头 |
| 经验回放 | `ReplayBuffer` | 存储真实经验，支持随机采样 |

### 模型网络结构

```
输入: [state, action] 拼接
         │
    ┌────┴────┐────────┐
    ▼         ▼        ▼
 状态预测头  奖励预测头  终止预测头
 (MSE Loss) (MSE Loss) (BCE Loss)
    │         │        │
    ▼         ▼        ▼
   ŝ'        r̂        d̂
```

### 训练流程

每一步交互后执行：

1. **Direct RL**：从经验池采样真实经验，更新 Q 网络
2. **Model Learning**：用同一批经验更新环境模型
3. **Planning**：重复 $n=10$ 次，从经验池采样 $(s, a)$，用模型预测 $(\hat{s}', \hat{r}, \hat{d})$，用模拟经验更新 Q 网络
4. **Target Update**：每 10 步硬更新目标网络

### 关键超参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `N_PLANNING_STEPS` | 10 | 每步规划次数 |
| `BATCH_SIZE` | 256 | 批量大小 |
| `GAMMA` | 0.99 | 折扣因子 |
| `LR_Q` | 1e-3 | Q 网络学习率 |
| `LR_MODEL` | 1e-3 | 模型学习率 |
| `EPSILON_DECAY` | 0.995 | $\epsilon$ 衰减率 |
| `UPDATE_TARGET_FREQ` | 10 | 目标网络更新频率 |

## 优缺点总结

> [!success] 优点
> - **样本效率高**：通过模型规划，每条真实经验的利用率大幅提升
> - **概念简洁**：框架统一，Direct RL 和 Planning 使用相同的更新规则
> - **可扩展**：规划步数 $n$ 可灵活调整，$n=0$ 退化为标准 Q-learning
> - **实现简单**：在标准 Q-learning 基础上只需添加模型学习和规划模块

> [!failure] 缺点
> - **模型偏差**：模型不准确时，规划会将错误传播到策略中（compounding error）
> - **环境非平稳**：默认不适应环境变化（需 Dyna-Q+ 改进）
> - **计算开销**：每步需额外训练模型和执行 $n$ 步规划
> - **确定性模型局限**：Tabular 版本假设确定性转移，不适用于随机环境
